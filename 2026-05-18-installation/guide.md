# VS Code + Claude Code 설치 가이드

## 결론 (TL;DR)

**비개발자도 Claude Code를 쓸 수 있도록 VS Code(책상) + Claude Code(비서)를 50분 안에 함께 세팅하는 강의입니다.**

이번에 만든 환경은 앞으로 22기 끌림 영상 스터디 전 과정에서 그대로 재사용됩니다.

- **이 페이지 한 장만 따라하면** Windows / Mac 어느 쪽이든 50분 안에 설치 완료
- **꼭 챙겨야 하는 2가지** ① VS Code 확장 프로그램 `Claude Code for VS Code`도 같이 설치 (오류 시 캡쳐+질문으로 자가 해결용) ② CLI 설치 후 **VS Code 완전 종료 후 재실행**, 그리고 `claude --version`으로 검증
- **두 가지 방식**으로 Claude를 부른다: ① VS Code 확장 프로그램 (GUI) ② 터미널 `claude` 명령 (CLI). 둘 다 깔아두면 입문자 친화 + 정석 둘 다 확보
- **설치는 한 번만**, 외울 필요 없음 — 다음 수업부터는 본격 실습

## 강의는 어떤 내용이었나

지피터스 22기 끌림 영상 스터디의 **첫 설치 세션**으로, 약 50분간 줌 라이브로 진행됐습니다. 스터디장 **타타**가 비개발자 입문자도 따라올 수 있도록 OS별(Mac → Windows) 화면을 번갈아 보여주며 다음 6단계를 함께 설치했습니다.

1. **VS Code** + 한국어 언어팩 + 라이트 테마 + 자동 저장
2. **폴더 열기** + 터미널 열기
3. **Claude Code for VS Code** 확장 프로그램 설치 + 로그인
4. **Homebrew** 설치 (Mac만)
5. **Git** 설치
6. **Claude Code CLI** 설치 + `claude --version` 검증

핵심 메시지는 *"확장 프로그램을 일부러 먼저 깔자 — 설치 중 오류가 나면 캡쳐해서 그 안의 Claude에게 물어보면 대부분 해결된다"*는 자가 해결 워크플로우였습니다.

## 0. 한눈에 보는 설치 순서

설치는 **이날 한 번만** 해두면 끝. 순서대로 6단계만 따라하면 OK.

1. **VS Code** 설치 → 한국어 언어팩 → 라이트 모던 테마 → Auto Save 켜기
2. **폴더 열기** + 터미널 새로 열기 (책상 완성)
3. **Claude Code for VS Code** 확장 프로그램 설치 + Subscription 로그인
4. **Homebrew** 설치 (Mac만, Windows는 건너뛰기)
5. **Git** 설치 (Mac: `brew install git`)
6. **Claude Code CLI** 설치 → VS Code 재시작 → `claude --version` 확인 → `claude` 실행 → "I trust this folder" → 테마 선택

> 6단계까지 끝나면 폴더 + 터미널 + 확장 프로그램 세 가지가 모두 갖춰진 "완벽한 상태" 완성.

---

## 1. 비유 — 책상과 비서

본격 설치 전에 강사가 강조한 비유:

| 도구 | 역할 |
|---|---|
| **VS Code** | 작업 책상 |
| **Claude Code** | 책상 옆에 앉은 똑똑한 비서 |
| **폴더** | 책상 위에 펼친 작업 공책 |

> Claude는 웹에서 채팅으로만 답을 주지만, **Claude Code는 내 컴퓨터 폴더에 직접 들어가서 파일을 만들고 고치고 실행할 수 있다**. 그래서 VS Code라는 책상에 비서를 앉혀두고 함께 일하는 구조.

Claude [Cowork] 같은 다른 형태도 있지만, 가장 확장성 있게 작업할 수 있는 건 Claude Code.

## 2. VS Code 설치

### 2-1. 다운로드

1. 구글에서 `VS Code` 또는 `Visual Studio Code` 검색
2. 공식 사이트에서 본인 OS에 맞는 인스톨러 다운로드
3. 설치 후 실행하면 초기 화면이 뜸

### 2-2. 한국어 언어팩 설치

영어 화면이 부담스러운 입문자를 위해 **가장 먼저** 한국어 언어팩 설치.

1. 왼쪽 사이드바의 **확장(Extensions, 블록 모양)** 아이콘 클릭
2. 검색창에 `Korean Language Pack` 입력
3. `Korean Language Pack for Visual Studio Code` 설치
4. 설치 후 우측 하단 **재시작(Restart)** 클릭

> 한국어가 안 되면 → VS Code를 완전히 종료 후 다시 열기.

### 2-3. 라이트 테마 (라이트 모던) — 선택

검은 화면이 부담스러우면 라이트 모드로 변경.

| OS | 메뉴 경로 |
|---|---|
| **Mac** | Code → 설정 → 테마 → 색 테마 |
| **Windows** | File → 설정 → 테마 → 색 테마 |

목록에서 **Light Modern**(라이트 모던) 선택.

### 2-4. 자동 저장 (Auto Save) 켜기

상단 메뉴 **File → 자동 저장** 토글 ON.

> 당장 필수는 아니지만 이날 같이 켜두면 편함.

---

## 3. 폴더 열기 + 터미널 열기

### 3-1. 폴더 열기

1. **File → 폴더 열기** (또는 시작 화면의 "Open Folder")
2. 새 폴더 아무거나 만들어서 열기 (오늘은 빈 폴더로 충분)

> 폴더를 여는 이유: AI 선생님(Claude Code)이 작업할 공간이 필요하기 때문. 앞으로의 모든 스터디 실습은 **책상 열기 → 폴더 열기 → 작업** 순서로 진행됨.

### 3-2. 터미널 열기

1. 상단 메뉴 **터미널 → 새 터미널** (Mac: `Ctrl + ` 백틱)
2. 아래쪽에 터미널 창 생성됨
3. (선택) 터미널 창을 드래그해서 우측에 배치하면 편함

> 이 단계에서 **"폴더 + 터미널 + (다음 단계의) 확장 프로그램"** 3종이 모두 보이는 화면이 강사가 말한 "완벽한 상태".

---

## 4. Claude Code 확장 프로그램 설치

VS Code 안에서 비개발자도 쉽게 Claude Code를 쓸 수 있게 만든 GUI.

### 4-1. 설치

1. 사이드바의 **확장(Extensions)** 아이콘 클릭
2. 검색창에 `Claude Code` 입력
3. **Claude Code for VS Code** (제작: Anthropic) 설치

### 4-2. 로그인

1. 설치 후 좌측 패널에 Claude Code 아이콘 등장 → 클릭
2. 로그인 화면이 뜨면 **Subscription**으로 로그인 (본인이 결제한 Claude 계정)
3. 브라우저로 인증 → VS Code로 돌아오기

### 4-3. 확장 프로그램의 진짜 용도

**터미널 CLI 설치 중 오류가 발생할 때 트러블슈팅 도구로 사용한다.**

> 강사가 일부러 확장 프로그램부터 깐 이유:
> 다음 단계인 CLI 설치는 변수가 많아서 입문자가 막히기 쉬움. 그때 오류 메시지를 캡쳐해서 확장 프로그램의 Claude에 붙여넣고 *"왜 안 돼? 알려줘"* 또는 *"해줘"* 라고 물어보면 대부분 해결책을 제시함.

---

## 5. Homebrew 설치 (Mac만)

> Windows 사용자는 이 섹션 건너뛰기.

### 5-1. 설치 명령

터미널에 Homebrew 공식 설치 스크립트 붙여넣기:

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

### 5-2. 비밀번호 입력

설치 도중 **Mac 로그인 비밀번호** 입력 요구 → 노트북 켤 때 쓰는 그 비밀번호 입력.

### 5-3. 검증

설치가 진행되는 동안 잠시 기다림. 끝나면 안내에 따라 **엔터 몇 번** 누르고:

```bash
brew --version
```

버전이 뜨면 완료.

---

## 6. Git 설치

### Mac

```bash
brew install git
```

설치 완료 시 맥주 이모지(🍺)가 출력되고 프롬프트에 `%`가 다시 나타나면 끝.

### Windows

Homebrew가 없으므로 강의 자료에 명시된 Windows용 방식(예: Git for Windows 인스톨러)으로 진행.

---

## 7. Claude Code CLI 설치

여기까지 오면 "선생님을 책상으로 부르는" 단계.

### 7-1. 설치

새 터미널을 열고 강의 자료에서 제공된 설치 명령을 복붙 (Mac/Windows 동일):

```bash
npm install -g @anthropic-ai/claude-code
```

> Windows의 경우 새 터미널이 **PowerShell**로 열림. 그 화면에서 그대로 실행.

### 7-2. VS Code 재시작

설치가 끝나면 **VS Code를 완전히 종료한 뒤 다시 열기**. (재시작하지 않으면 `claude` 명령을 인식 못 함)

### 7-3. 버전 확인

새 터미널에서:

```bash
claude --version
```

버전 번호가 출력되면 설치 성공.

> 버전이 너무 낮게 뜨면 확장 프로그램의 Claude에게 *"내 버전은 왜 이렇게 낮아?"* 라고 물어보면 됨.

---

## 8. Claude Code 첫 실행

### 8-1. `claude` 실행

터미널에서:

```bash
claude
```

### 8-2. 첫 화면 분기

**"이 폴더를 신뢰합니까?"** (`Do you trust this folder?`)
→ **Yes / 믿습니다** 선택

이후 **테마 선택** 화면이 뜨면 본인 취향에 맞게:
- 검은 화면이 부담스러우면 **Light / 라이트 모드**
- 눈이 피곤하면 **Dark / 다크 모드** (기본값)
- 그 외 모든 옵션은 **Recommend / 추천**으로 진행

### 8-3. 대화 테스트

프롬프트에 한국어로 인사:

```
안녕?
```

Claude가 대답하면 설치 성공.

> 로그인 화면이 뜨면 안내대로 **Subscription**으로 인증.

---

## 검증 체크리스트

다음 3가지가 모두 정상이면 설치 완료:

- [ ] VS Code에서 **폴더 → 터미널 → Claude 확장 프로그램 아이콘**이 모두 보임
- [ ] 터미널에서 `claude --version` 입력 시 버전 번호 출력
- [ ] 터미널에서 `claude` 실행 후 한국어로 인사하면 Claude가 답변

## 자주 막히는 곳

| 증상 | 해결 |
|---|---|
| 한국어 언어팩 적용이 안 됨 | VS Code 완전 종료 → 다시 열기 |
| `claude --version`이 안 됨 | VS Code 종료 → 다시 열기 → 새 터미널에서 재시도 |
| `claude` 첫 실행이 멈춤 | 터미널 종료 → 다시 열고 `claude` 재실행 |
| 오류 코드가 빠바박 뜸 | **캡쳐 → Claude Code 확장 프로그램에 붙여넣기 → "왜 안 돼?" 또는 "해줘"** |
| 명령어가 안 떠요 | 설치가 진행 중일 수 있음. 잠시 대기 후 재확인 |
| Mac 스크린샷 + 클립보드 복사 | `Cmd + Shift + 4` 누른 뒤 `Ctrl` 누른 채로 드래그 → 클립보드에 캡쳐 → `Cmd + V`로 붙여넣기 |

> 혼자 해결이 안 되면 지피터스 네트워킹 방에서 `@타타` 태그하고 캡쳐 + 상황 설명 올리기.
