결론 (TL;DR)

클로드 코워크 = 클로드가 내 컴퓨터의 "지정 폴더"에 직접 들어와 파일을 만들고 정리하는 도구. 채팅(바닐라)과 클로드 코드의 중간. 프로 이상이면 쓸 수 있고, 코딩 몰라도 자연어로 결과물을 만든다.

정식 강의가 아닌 실습·학습 세션. 강사가 미리 준비한 zip 자료(mesh·영수증·미팅노트 폴더)를 받아, 참가자들이 클로드 코워크에 폴더를 지정하고 직접 정리·생성을 돌려보는 핸즈온 위주였다. 후반부엔 시간 관계상 웹 자동화(Claude in Chrome + 플레이 MCP)를 설명으로 대체했다. (상세 방법은 강사가 배포한 PDF에 수록.)

0. 한눈에 보는 코워크 실습 흐름

  1. 클로드 데스크탑 앱 설치 (Win/Mac) — 웹 아님, 프로 이상 결제
  2. 코워크 진입 → 왼쪽 하단 폴더 아이콘
  3. 작업 폴더 지정 → "허용" 클릭
  4. 프롬프트 입력 (구체적으로 3~4줄)
  5. 승인하며 진행 → 결과물 확인

1. 클로드 코워크란?

일반적인 클로드 채팅은 질문하고 답을 받는 대화형이다. 코워크는 한 단계 진화해, 지정한 로컬 폴더에 직접 접근한다. 그래서 클로드가 파일을 직접 조작(편집·생성)해 결과물을 만들 수 있다.

  • 클로드 채팅 = 클로드 바닐라(아이스크림 기본맛처럼 "오리지널"이라는 코딩 용어).
  • 바닐라 → 코워크 → 클로드 코드 순으로, 코워크는 중간 역할.
  • 채팅·답변에 그치지 않고 실제 사무·업무·생활에 다양하게 쓸 수 있는 게 코워크.

클로드 코드가 무섭다면 코워크부터. 데스크탑의 코워크만으로도 웬만한 PC 작업을 자동화할 수 있고, 익숙해지면 클로드 코드로 넘어가기 쉽다.

2. 사용 전 3원칙 + 설치

3원칙

  1. 전용 작업 폴더를 따로 만든다 / 지정한다. 바탕화면 등 전체 폴더에 권한을 주면 원치 않는 변형·삭제가 일어날 수 있다.
  2. 복사본으로 작업한다. 직접 수정·삭제가 가능하므로 중요한 원본은 따로 보관.
  3. 권한은 "허용", "모든 승인 건너뛰기"는 하지 않는다. - 항상 허용: 계속 안 물어봄 (편함, 위험) - 허용: 동작마다 한 번씩 물어봄 (안정적) ← 기본 권장

설치

  • 클로드 데스크탑 앱을 Win/Mac용으로 다운로드 (웹사이트 사용 X).
  • 프로 이상 결제 필수.
  • 코덱스로는 어렵다 — 코워크는 내 컴퓨터를 다루는 기능에 특화.

폴더 지정 방법

왼쪽 하단 폴더 아이콘 → 다른 폴더 선택 → 폴더 위치로 이동 → 폴더 선택 → 허용. (프로/맥스가 아니면 코워크 영역이 안 보이고 챗만 나올 수 있다. 이 방식은 클로드 코드의 폴더 지정과 동일.)

3. 실습 1 — 폴더 정리 (mesh 폴더)

zip 안의 mesh 폴더(파일이 중구난방)를 지정하고, 제공된 프롬프트로 파일명을 정리한다.

  • 파일명 변경 정도는 추론 능력이 크게 필요 없으니 하이쿠(Haiku)로 충분.
  • 시간은 다소 걸린다. 지금은 테스트지만, 실제로는 여행 사진을 날짜별·종류별로 분류하는 등 사람이 일일이 하기 힘든 일을 한 번에 처리할 수 있다.

코워크는 "분류"보다 결과물 생성(PPT·엑셀 등)이 주 용도. 클로드 코드는 지침서를 읽고 코딩하지만, 코워크는 자연어로 생성물을 만든다.

4. 실습 2 — 영수증 OCR 정리 (receipt 폴더)

receipt 폴더로 바꿔 지정 → 제공 프롬프트로 실행. 사진을 OCR(이미지를 읽는 기능)로 읽어 보고서·표로 정리한다.

  • 자영업자·경리 없는 소규모 회사, 종합소득세 신고(5월)에 유용.
  • 코워크는 내 컴퓨터 안에서 작업하므로 업로드·다운로드 같은 중간 과정이 없다.

OCR 성능 체감(강사 의견): GPT 5.4 이미지 2 [?원문] > 클로드 순. 제미나이 3.5 프로가 나오면 달라질 수도.

"정리해줘"는 위험 — 구체적으로 써라

한국어는 모호해서 클로드가 의역하면 큰일 날 수 있다. - 나쁜 예: "바탕화면 정리해줘" - 좋은 예: "○○ 기준으로 분류해줘 / ○○로 정리해줘 / ○○를 지정해 삭제해줘" - 귀찮아도 3~4줄로. (정리는 되돌릴 수 있어도 삭제는 못 되돌린다.)

5. 실습 3 — 미팅 노트 정리

미팅 노트 폴더를 지정해 진행. 방법은 배포된 PDF에 수록. 회의록 정리에 매우 유용(다만 속도는 빠르지 않음).

사진 메타데이터 활용

  • 사진의 GPS·날짜·시간 메타데이터를 켜고 찍으면 "프랑스에서 찍은 사진만", "특정 좌표만"처럼 정리 가능.
  • 카카오톡으로 공유한 사진은 메타데이터가 날아간다 → 내 오리지널 사진 필요.
  • 얼굴 지정 시 사람별 정리도 가능(단, 사진 많으면 토큰·시간 많이 듦).

파일 편집의 한계

  • 워드·PPTX·엑셀은 편집이 아니라 새 파일 생성 방식이라 결과가 완벽하지 않을 수 있음.
  • 마크다운(.md)·JSON 등 코딩 친화 파일은 내부를 직접 수정·편집한다. (옵시디언 정리 이야기가 나오는 이유.)

6. (심화 · 설명) 웹 + 메신저 연결 자동화

시간 관계상 실습 대신 설명. 방법은 PDF에 수록.

Claude in Chrome

  • 크롬 ⋮ → 확장 프로그램 관리 → Claude in Chrome 설치.
  • 클로드가 크롬을 제어해 웹서핑·실시간 정보 접근·웹 크롤링·반복 워크플로우(자동화)를 수행.
  • 프로 이상. 엣지도 가능(사파리는 X).
  • 사용법: 코워크/클로드 코드에서 검색·웹 확인이 필요하면 자연어 명령 → 클로드가 알아서 호출.

플레이 MCP [?플라이 MCP]

  • 카카오가 운영, 카카오·네이버 연결. 크롤링/검색 결과를 카카오톡 "나에게 보내기"로 보고하게 가능.
  • 유튜브에 나오는 요새 AI 에이전트들의 기본 원형. 쓰다 보면 에이전트·봇·크롤링 자동화에 익숙해진다.

API 장벽 & 실시간 웹서핑

  • 인스타·X는 API가 막거나 유료. 카카오·네이버는 무료 잔고가 있어 일부 정보 수집 가능.
  • 실시간 웹서핑 기본 가능 LLM은 구글(제미나이)·퍼플렉시티 정도. Claude in Chrome 설치 시 클로드도 실시간에 근접.
  • MCP는 할루시네이션이 거의 없다 — "출처를 밝히라"만 추가하면 됨. (예: 카카오맵 MCP → 주소 입력 시 가까운 지하철역 입구 탐색.)

7. 마무리 — 강사의 워크플로우 팁

  • 오늘 자료는 모두 PDF에 수록. (학습반장 수업이라 다시보기가 없을 수 있으니 PDF 참고.)
  • 이 PDF 자체도 클로드 코워크로 제작. 핵심 레시피:

① 초안은 제미나이로 (유튜브 접근 권한이 있어 영상 요약이 되는 유일한 LLM) ② 보강·PPT 제작은 클로드 코워크로 → 유튜브 요약본을 제미나이로 만들고, 그걸 코워크에 넘겨 PPT를 만들면 퀄리티가 굉장히 좋다.
