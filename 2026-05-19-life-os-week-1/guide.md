## 결론 (TL;DR)

> **"판단은 사람이, 실행은 AI가, 기억은 시스템이." 빈 옵시디언 볼트 하나 + 매일 돌리는 스킬 2개(투데이·클로즈데이) + 북극성 타이머 — 이게 60일 만에 AI 조직으로 자란 인생 OS의 전부다.**

### 인생 OS 3원칙

| # | 원칙 | 뜻 |
|---|---|---|
| 1 | **판단은 사람이 한다** | AI에게 판단을 떠넘기지 않는다 |
| 2 | **AI가 실행하게 한다** | 반복·실행은 위임 |
| 3 | **시스템이 기억하게 한다** | "그거 어디 있지?"를 시스템이 답한다 (가장 중요) |

### 오늘 챙겨야 할 5가지

| # | 메시지 | 한 줄 |
|---|---|---|
| 1 | **빈 폴더에서 시작** | 거창한 앱·조직 아님. 데일리노트 + 포커스 타이머 + 스킬 2개로 출발 |
| 2 | **목표 ≠ 행동** | 타임 트래킹이 "마케팅 최우선"이라 말하면서 강의 준비만 하던 간극을 드러냄 |
| 3 | **사고가 직원을 만든다** | 마케팅 사고→조시, 영상 비용 사고→봉 감독, SMS 사고→사피엔. 반복 문제를 역할로 위임 |
| 4 | **부채 청산 루프** | 미루는 진짜 비용은 "다시 꺼내는 심리적 비용". 한 건 풀면 다음이 쉬워진다 |
| 5 | **북극성 지표** | 4~8주간 향할 단 하나의 목표. 숫자 필수, 내 노력으로 가능한 것 |

### 1주차에 실제로 할 일

| 단계 | 무엇을 |
|---|---|
| 설치 | 안티그라비티/VSCode → 터미널에서 인생 OS 초기화 스킬 설치 → 볼트 생성 |
| 옵시디언 | "보관함 폴더 열기"로 `라이프OS_<영문닉>` 볼트 열기 (생성X) |
| 운영 | 내일부터 아침 **투데이**, 하루 1시간+ **북극성 타이머**, 저녁 **클로즈데이** |
| 과제 | 일주일(최소 3일) 운영 기록 → 사례 게시글 → **다음 주 화요일 발표** |

> **주의:** 강사가 패키징에서 `투데이`·`클로즈데이` 스킬을 빠뜨려, 저녁에 재배포·단톡방 공지 예정. 받으면 같은 방식으로 볼트에 한 번 더 스킬 추가.

"무언가를 만드는" 다른 스터디와 달리, 인생 OS는 **여정을 기록하는 스터디**입니다. 도구가 늘수록 오히려 복잡해지는 역설을, 사람·AI·시스템의 역할을 나눠 푸는 것이 목표입니다.

| 부 | 섹션 | 핵심 |

| 3 | 북극성 지표 | 단일 목표 + 숫자 + 통제 가능성, 북극성 타이머 |
| 4 | 실습·과제 | 설치 6단계, 일주일 운영 → 게시글 → 화요일 발표 |

## 1. 도구는 많아졌는데 왜 더 복잡할까

챗GPT·클로드·제미나이·코덱스·클로드 코드에 자동화 도구까지 — 도구는 많아졌는데 하루는 더 정리되지 않습니다.

- **답은 많아졌는데 결정은 더 어려워진다.** 선택지가 늘수록 고르는 부담이 커진다.
- **기억이 흩어진다.** "어제 코덱스한테 시킨 거 어디 있지?" 너무 많이 만들어 그 속에 숨어버린다.

해법이 3원칙입니다(위 TL;DR). 특히 **시스템이 기억하게 한다** — 이제 폴라에게 "그거 어디 있지? 찾아줘" 하면 끝납니다.

> 옵시디언 노트는 적어두고 돌아보는 사람이 거의 없습니다(작년 그날 일기를 꺼내보지 않듯). 인생 OS는 "익히는" 게 아니라 **"돌아가게"** 만드는 것. *"모든 시스템은 빈 폴더 하나에서 시작된다."*

---

## 2. 작은 시작 — 빈 볼트와 스킬 2개

처음 만든 건 단 네 가지: **데일리노트, 포커스 타이머**, 그리고 스킬 **투데이 / 클로즈데이**.

| 스킬 | 시점 | 하는 일 |
|---|---|---|
| **투데이** | 아침 | 오늘 이슈 정리 + 어제 포커스 타이머 기록으로 집중도 요약 |
| **클로즈데이** | 저녁 | 하루 일과 정리 + AI 회고 작성 |

이 두 스킬을 매일 돌리는 것만으로 기록이 쌓이고, 8주쯤 지나면 조직이 됩니다. 단, 조직이 커진다고 모든 걸 자동에 맡기는 건 아니고 — 오히려 **위임과 핸드오프**가 중요해집니다.

### 목표와 행동의 불일치

처음 목표엔 "마케팅 최우선(매출!)"이라 적었지만, 타임 타이머 실제 기록은 강의 준비·시스템화·자동화에 쏠려 있었습니다. 인생 OS가 처음 해준 일은 바로 **이 간극을 찾아준 것**. 그래서 우선순위를 바꿉니다.

---

## 3. 사고가 직원을 만든다 — 에이전트 조직

에이전트는 거창한 계획이 아니라 **사고가 날 때마다** 하나씩 태어났습니다.

| 에이전트 | 계기(사고) | 역할 |
|---|---|---|
| **폴라리스(폴라)** | 4/9 이름 부여 | COO — 일과 삶 운영, 메모리 시작 |
| **조시** | 마케팅 자동화 사고 | 마케팅 전담 + 사실 검증 (외부 메시지는 틀리면 치명적) |
| **봉 감독** | 영상 자동화 비용 폭증(하룻밤 Veo3 API 8~9만 원 ×100회) | 영상 작업 + API 비용 게이트 |
| **봉 PD** | — | 봉 감독의 동생, 오픈클로에서 24시간 영상 작업 |
| **사피엔** | 무료 SMS 발송 사고 | SMS 발송 + 수강생 전체 관리 |
| **폴리·코코·채토끼** | 분산 테스트 통합 | 폴라가 일괄 운영 |

핵심: **반복되는 문제를 역할로 위임**한다. 혼자 다 하지 않는다.

### 핸드오프 — 보이지 않는 시스템

폴라가 일을 분리할 때 **핸드오프 문서**(인수인계)를 작성하면 새 에이전트가 컨텍스트를 이어받습니다. 단, 핸드오프는 **요약**이어야 합니다 — 너무 많이 담으면 컨텍스트 창이 꽉 차 전달이 안 됨. 그래서 모든 에이전트는 **단일 폴더에 매일 로그**를 남기고, 핸드오프엔 "어느 로그에 상세가 있다"는 요약만 넘깁니다. 클로드가 컨텍스트를 놓칠 때 복구 수단이 바로 이 핸드오프·메모리입니다.

## 4. 부채 청산 · 디데이 — 사람을 변화시키는 시스템

### 부채 청산 자기 강화 루프

> *"미루는 일은 시간이 부족해서가 아니라, 그 일을 다시 꺼내는 데 드는 심리적 비용이 매일 무거워지기 때문이다."* — 폴라 (D59 에세이)

4일 미룬 일이 막상 손대니 1시간도 안 걸렸습니다. 이어 "박스 치고"(1시간 동안 그것만 전념) 미루던 법인 공인인증서까지 끝내니, 막혀 있던 일들이 줄줄이 풀렸어요.

**한 건을 청산하면 다음은 더 쉬워진다. 부채가 쌓이면 다음은 더 어려워진다.** → 오늘 강의 끝나고 미루던 일 하나를 박스 쳐서 끝내보세요.

### 디데이 산출 폭증

디데이가 다가올수록 집중력이 강화되고 많은 일이 정리됩니다. 디데이는 생산성·자기계발 도구가 아니라 **사람을 변화시키는 시스템**입니다. 비싼 다이어리는 연중반이면 안 쓰게 되지만, 노트 + 시스템(투데이·클로즈데이 + 북극성 타이머)은 명확합니다. 변화는 가장 어렵고, 시스템이 아니면 사람은 변하기 어렵습니다.

## 5. 북극성 지표 (North Star Metric)

4~8주간 향할 **단 하나의 목표**. 워크북·코치 잼·타이머 앱 링크가 제공됩니다.

| 규칙 | 설명 | 예 |
|---|---|---|
| 단일 목표 | 하나만 정해 바라보며 간다 | — |
| 숫자 필수 | 측정 가능해야 | "월 매출 3000만 원" O |
| 통제 가능 | 내 노력으로 도달 가능 | "유튜브 구독자 10만" X (불가항력) |

> 폴라리스 = 북극성. 그래서 폴라리스 OS, AI "폴라"도 여기서 나왔습니다.

### 북극성 타이머 핵심

- **4모드**: 10분 집중 / 20분 딥포커스 / 직접 설정 / 휴식 (추천: 일반 10분, 딥포커스 20분)
- 플레이 전 "목표 관련 일"이면 체크 → 목표 연동. 투두 3개까지 등록
- **내보내기**: 오늘/어제, 그리고 일주일(목표 비율·주간 분석·CSV)
- 발견: **집중해서 일하는 시간은 생각보다 적다** (강사도 오늘 15세션=150분, 이번 주 목표 관련 47%)

## 6. 실습 — 인생 OS 설치 6단계

> 따라 하기 어려우면 집중해서 보고 내일 녹화본으로. 토요일 오프 모임에서도 잡아드림. 스트레스 받지 말 것.

| 단계 | 할 일 | 주의 |
|---|---|---|
| 1 | 안티그라비티/VSCode에서 새 폴더 생성·열기 → 터미널 열기 | 클로드 코드/코덱스 미설치여도 OK |
| 2 | 터미널에 설치 명령어 붙여넣기(윈도우=PowerShell / 맥) → 엔터 | **PowerShell은 관리자 권한**으로 |
| 3 | "라이프 OS init 실행" 문장 입력 → 영문 이름 입력 → 볼트 생성 | 안티그라비티/코덱스는 슬래시 대신 키워드 문장 |
| 4 | 옵시디언 설치 → **"보관함 폴더 열기"** → `라이프OS_<영문닉>` 선택 | "생성" 누르면 X |
| 5 | 그래프뷰로 5개 파일 확인, **화면 캡처** | 북극성 정의·첫 노트·워크/라이프 컨텍스트·CLAUDE.md |
| 6 | 북극성 타이머 한 세션 → 내보내기 → 볼트 데일리 폴더에 저장 | **쓰는 컴퓨터에서** 다운로드 |

## 자주 막히는 곳

| 증상 | 대응 |
|---|---|
| 설치 명령어가 안 먹음 | WSL 설치 가이드대로 먼저 설치 |
| "가상화가 꺼져 있어 시작되지 않음" | BIOS에서 가상화 활성화 → 저장·재부팅 → 재시도 |
| 환경별로 제각각 에러 | 화면 캡처해 **퍼플렉시티**에 질문("한글로 말해" 추가) |
| 볼트가 이상함 | "생성" 말고 "보관함 폴더 열기"로 `라이프OS_닉네임` 폴더를 열었는지 확인 |
| 클로즈데이/투데이 스킬 없음 | 강사 패키징 누락 — 재배포본 받아 볼트에 스킬 한 번 더 추가 |

## 7. 첫 과제

- **인생 OS를 일주일(최소 수·목·금 3일) 운영하기.** 무언가를 만드는 게 아니라 운영·기록.
- 매일: 아침 **투데이** → **북극성 타이머 1시간+** → 내보내기(데일리 폴더) → 저녁 **클로즈데이**.
- 운영 경험을 사례 게시글로 정리 → **다음 주 화요일 발표**(첫 발표라도 무조건. 청강생은 의무 아님).
