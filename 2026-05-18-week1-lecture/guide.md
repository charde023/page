## 결론 (TL;DR)

**"AI에게 대체될 것인가, AI를 동료로 만들 것인가" — 그리고 그 동료에게 일을 시키려면 디자인을 어떻게 시각화할 것인가.**

22기 1주차 핵심 강의는 두 개의 50분 강의로 구성됐습니다. 정균님은 *왜* AI 에이전트로 일하는 방식이 바뀌어야 하는지, 인준님은 *어떻게* AI 도구로 빠르게 디자인하고 커뮤니케이션할지를 다뤘습니다.

- **AI는 3단계로 진화 중** ① 챗봇(2022) → ② 바이브 코딩(2024) → ③ **에이전트(2025~, 도구+메모리를 가진 동료)**
- **상시 동료를 만들려면**: **OpenCloud**(팀·여러 도구 통합) 또는 **Hermes Agent**(혼자 반복 작업 자동화) — 둘 중 하나 골라 4주간 본인 동료 구축
- **영업비밀은 기록**: 암묵지를 디지털화해야 AI가 *"나처럼"* 일함. 메모장·노션·옵시디언 어디든. **예전 기록 절대 버리지 말 것**
- **디자인은 Claude Design으로 시작**: 비디자이너도 PRD만 주면 시안 + 트윅스 옵션 자동 생성. 단, **개발 도구 아님** — 디자인 시스템 추출 후 펜슬·Claude Code로 구현 연결
- **도구는 상황에 맞게**: Claude Design(시각화), AI Studio(MVP까지), 펜슬(디자이너 친화), Stitch(모바일)
- **Just Do It** — 완벽주의자는 살아남기 어려운 시대

## 1. 운영 OT 핵심 (다이님)

### 1-1. 학습 시스템 (필수 즐겨찾기)

22기부터 모든 정보가 **AI 스터디 전용 학습 시스템**에 통합됨.

- **접근**: 지피터스 홈페이지 → 우측 상단 프로필 → "내 AI 스터디"
- **포함**: 줌 링크, 단톡방, 공지사항, 다시보기, 캘린더, 스터디 제출
- **다시보기**: 매 강의 다음날 오전 1주차 폴더에 업로드
- **GP 사전**: 운영 용어 정리 (공동 스터디장, 덕후방 등)

### 1-2. 운영 비서 "뽀짝이"

지피터스 운영을 보조하는 **에이전트 비서**.

- 카톡방 답변, 문자 발송, 슬랙 자동 처리
- 24개 단톡방 운영
- 네트워킹 방에서 직접 질문 가능

### 1-3. 스케줄 & 정책

| 일정 | 내용 |
|---|---|
| 오늘·다음 주 월요일 | 핵심 강의 |
| 내일~수요일 | 스터디별 OT (실습 따라하기 — **노트북 필수**) |
| **금요일 15시까지** | 수강 변경 가능 (보통 하루 5~6건 자리 바뀜 — 대기 중이면 기다릴 것) |
| 토요일 | 오프라인 모임 (~300명 예상, **사전 신청 필수**) |
| 2~4주차 | 사례 게시글 발표 |

**추가 모임**: 5/30 역삼역 모각, 6/6 비어룸 맥주 모각 (다음 주 별도 공지)

> **참여 권장**: 핵심 강의 + 본인 수강 4번 = 필수. 청강은 *1개만 제대로*. 카톡방 평균 80명 → 정보 홍수 주의.

### 1-4. 첨강

- 별도 신청 없음
- 학습 시스템 홈 → 줌 입장하기로 그냥 입장
- 전체 스터디 메뉴 → 단톡방 입장 자유

## 2. 정균님 강의 — AI 에이전트 시대

### 2-1. 강사

**박정균**. 7기부터 지피터스 활동, 22기 *"에이전틱 AI로 투자 활용하기"* 스터디장. 제이엠얼 시스템 AXTF팀 팀장. 엠파레 앰배서더. 패스트 캠퍼스 엠파레 강의 (최다 판매), SK 하이닉스·삼성전자·LG 임원 교육 다수.

### 2-2. 도입 — "사람 없는 책상"

회사 책상에 **사람 없이 Claude만 돌아가는 사진**으로 시작. OpenCloud, 나노클로, LLMs 에이전트가 무수히 돌아가는 시대.

> *"공포스럽지만 저항하면 안 됩니다. 막을 수 없는 흐름이니 받아들이세요."*

### 2-3. AI ↔ 인간 관계의 3단계 진화

| 단계 | 시기 | 대표 도구 | 핵심 |
|---|---|---|---|
| **챗봇** | 2022 | ChatGPT 3.5 터보 | 질문 → 답만. 할루시네이션 심함. *"엑셀 정리해줘"* → *"죄송합니다"* |
| **바이브 코딩** | 2024 | Lovable, Cursor | 결과물이 손에. *"하버드 졸업 인턴"* — 만들긴 하는데 잘 안 돌아감. POC 좋음, 프로덕션 불안. 보안 노출 사례 多 |
| **에이전트** | 2025~ | Claude Code, Codex, Gemini CLI | **LLM + 도구 + 메모리.** 셰프처럼 주문·요리·설거지 한 사이클을 끝까지. 내 PC 파일을 직접 읽고 작업. Human-in-the-loop |

> **핵심 메시지**: *"도구가 바뀐 게 아니라 AI와 인간의 관계가 바뀌고 있다."*

### 2-4. 에이전트의 정의

- **챗봇(LLM)**: 텍스트 in → 텍스트 out. 도구 X. *"이 코드 너가 실행해봐"*
- **에이전트**: LLM이 **도구를 쥐었고 메모리를 가졌음**. 직접 실행 → 결과 → 보고. *"슬랙 발송 완료, 추 주세요"*

도구 사용 예: 웹 검색(크롬 자동 띄움), 슬랙 MCP, 지메일 읽기/초안/발송. MCP = "USB-C 타입처럼 SaaS 도구를 AI에 꽂는 표준".

### 2-5. 에이전트 사례

#### Claude Code로 바탕화면 정리

스크린샷·PPTX·정산 엑셀·메모·PDF·사진이 뒤섞인 바탕화면 → *"정리해줘"* → 폴더에 종류별로 깔끔하게 분류. **Mac·Windows 둘 다.**

#### Claude Cowork — 비개발자용 GUI

Anthropic이 비개발자를 위해 만든 Claude Code의 GUI 버전.

1. 프롬프트 입력
2. AI가 *"요약? 디테일 노트? 다른 액션?"* 으로 의도 명확화
3. 플랜 → 투두 리스트 자동 생성
4. **병렬로 일 처리**
5. PPT·엑셀 내장 스킬 (업로드 없이 *"이번 달 매출표 만들어줘"* → 실제 파일 생성)

### 2-6. 상시 동료 시대 — OpenCloud vs Hermes Agent

24시간 옆에 있는 동료. 비교:

| | **OpenCloud** | **Hermes Agent** |
|---|---|---|
| **장점** | `soul.md`로 영혼 부여 (캐릭터 말투·성격). 슬랙·노션·메일·캘린더·디스코드 통합. 내장 스킬 풍부 | 쓸수록 나에게 최적화 (알잘딱깔센). 가볍고 안전. 자체 패턴 발견·플레이북 갱신 (INTJ 같은 친구) |
| **추천** | 팀·여러 도구 통합 봇 (예: 뽀짝이) | 혼자 반복 작업 오래 자동화 |
| **단점** | 약간 무거움 | 다른 사람과 공유 어려움 |

> 강사 코멘트: "현재 OpenCloud → Hermes로 토큰 사용량 무게중심이 이동 중. 둘 다 써보길 추천."

> ⚠️ **컴퓨터를 끄면 동료가 죽습니다.** 24/7 켜둬야 함.

#### 뽀짝이 사례 (OpenCloud)

- `soul.md`에 성격(고양이 말투) + 메모리 + M8 워크플로우 + 노션 컨텍스트
- `heartbeat` + `cron` 기능으로 새벽·주말도 작동
- Claude Max 200달러 계정 **2개** 사용

### 2-7. 영업비밀 — 기록

> **다이님이 뽀짝이를 잘 쓰는 비밀 = 기록.**

- **암묵지를 디지털화하라.** 상무님 머릿속 엑셀 시트 설계 의도, 본인 일하는 패턴 — 다 기록해야 LLM이 *"아, 나처럼"* 일함.
- 기록 없으면 매번 처음부터 컨텍스트 주입 → **토큰만 잡아먹음** (퀴즈 정답).
- **어디든 OK**: 메모장, 노션, **옵시디언** (강사 추천). 마이그레이션은 Claude Code가 다 해줌.
- ⚠️ **예전 기록 절대 날리지 말 것** — 하드디스크, USB 다 보관.
- **직접 타이핑이 1순위**, AI와 잡담하며 기록이 2순위. *"하루에 한 글씩 쓰세요."*

### 2-8. 역사 비유 + Just Do It

| 기술 | 초기 반응 | 결과 |
|---|---|---|
| 증기기관 | "광산에서 물 퍼내는 비싼 기계" | 새로운 판(공장·도시·운송)을 깐 사람이 부를 쥠 |
| 인터넷 | "학자들의 장난감, 팩스가 더 빠른데?" | 지금 안 쓰는 사람 없음 |
| **AI** | "이거 챗봇 아니야?" | **같은 패턴 — 새로 설계하는 사람이 큰 부를 쥘 것** |

> *"도구가 아니라 **구조의 변화**로 봐야 합니다. 기존 일에 얹지 말고 **아예 새로 설계**하세요."*

> **Nike: Just Do It.** 완벽주의자는 지금 시대에 살아남기 어렵습니다. 4주 동안 OpenCloud 또는 Hermes Agent로 **본인의 동료를 만드는 것**을 목표로 하세요.

## 3. 인준님 강의 — Claude Design 초보자

### 3-1. 강사

**황인준**. 팝업 스튜디오 근무. 지난 기수 *"주 4회 빌드"* 스터디장 (주 4회 웹 서비스 한 개씩 빌드).

> 개발자인데 왜 디자인 강의? — 본인의 바이브 코딩 결과물이 일관성 있다는 평을 자주 받음. 그 노하우의 핵심이 **디자인 시스템 운영**이라 공유.

### 3-2. 강의 한 줄

> *"코딩도 안 해보고 디자인도 안 해본 사람이, 짧은 시간 안에 본인이 만들 것을 시각화해서 소통할 수 있게 만드는 방법."*

### 3-3. 디자인 워크플로우의 3단계 진화

| 단계 | 도구 | 특징 |
|---|---|---|
| **전통적** | 피그마 + 개발자 | 디자이너 ↔ 개발자 분리. *"왜 피그마대로 안 했냐"* 가 갈등의 시작점 |
| **AI 코드 빌더** | Lovable, V0, Replit | 프롬프트로 동작 코드 + 디자인 한 번에. POC 좋음. 깊어지면 수정 어려움, 일관성 무너짐 |
| **AI 디자인 협업** | **Claude Design**, 펜슬 | 코드 X, **디자인 파일** O. 디자이너 워크플로우 그대로. 코멘트·공유 지원. 비디자이너도 시각화 가능 |

### 3-4. Claude Design 진입

| 항목 | 내용 |
|---|---|
| **위치** | **claude.ai 웹 버전 전용** (데스크탑 앱 메뉴에는 없음) |
| **버튼** | 좌측 사이드바 "디자인" |
| **생성 종류** | 새 프로토타입 / 디자인 시스템 / 와이어프레임 / 슬라이드 덱 |
| **템플릿** | 내 작업물 / Claude 제공 템플릿 / 내 디자인 시스템 기반 |

### 3-5. 실습 — 이탈리아 여행 플래너 리디자인

PRD(Product Requirements Document)를 첨부하면 그 명세대로 디자인. 강사 본인의 실제 이탈리아 여행 일정 관리 앱이 예시.

#### 입력 옵션

- **디자인 시스템**: 사전에 잡아둔 톤
- **스크린샷·코드베이스**: 기존 화면 참조
- **피그마 파일** 직접 임포트
- ⭐ **GitHub 레포 연동**: Claude Design의 **차별점**. 기존 GitHub 프로젝트 가져와 리디자인 (반대 방향 export는 X)

#### 옵션 선택

- 톤·타입 (예: 이탈리아 웜 + 세리프)
- 이모지·이미지 사용 X (토큰 절약 팁)
- 언어
- **옵션 개수**: 메인 1 + **트윅스**(여러 변종 자동 생성)
- 타겟 디바이스 (데스크탑/모바일)

#### 결과물 특징

- 실제 이탈리아 지도 클릭 가능. 일정별 마커. 트립 문서까지 포함
- 각 시안에 **코멘트** 가능 (피그마식 협업)
- **이력 자동 저장** → 롤백 가능
- 디테일 수정: 색·폰트·간격·테두리 등 요소 선택 후 직접 변경

### 3-6. 토큰 한도 ⚠️

> Claude Design은 **별도 주간 한도**. Claude Max / Pro / Enterprise 계정과 **무관**한 별도 토큰. 다 쓰면 별도 구매 필요. **이게 1순위 단점.**

### 3-7. 도구 비교 — 언제 무엇을 쓰나

| 도구 | 강점 | 추천 상황 |
|---|---|---|
| **Claude Design** | 초기 컨셉 시각화 + 커뮤니케이션 | 말로 설명 불가능한 아이디어 빠르게 보여주기 |
| **Google AI Studio** | "한 단계 지난 Lovable" — 디테일 + 의도 못 한 발견. GitHub export → Claude Code 이어가기 | MVP까지 동작 + 실제 배포 |
| **펜슬** | 디자이너 친화. 화면별 디자인 파일 → CLI로 정확 구현 | 디테일한 화면 설계, Claude Code 연동 |
| **Stitch** | 모바일 최적 | 모바일 앱 디자인 |

> ⚠️ **Claude Design은 개발 도구가 아닙니다.** 디자인 깎는 도구일 뿐. 코드까지 빨리 가려면 AI Studio.

### 3-8. 잘 쓰는 법

1. **구체적 묘사**: *"A, B, C, D가 완료되면 끝"* 식으로 명확하게
2. **하나씩 요청**: 안 풀리면 한 번에 하나만
3. **레퍼런스 첨부**: 시각화 정확도 ↑

### 3-9. Q&A 핵심

| 질문 | 답 |
|---|---|
| 디자이너 + 개발자 환경에서도 의미? | **YES.** 디자인 도구이자 **커뮤니케이션 도구**. 디자이너 빠른 초안 → 피그마로 옮기기, 개발자 *"디자인 수정 요청"* 즉시 시안화 |
| 기존 웹 UI 가져와서 바꿀 수 있나? | YES. PC 디자인 → 모바일 자동 변환 |
| 신규엔 약하고 리디자인엔 강해 보임 | 디자이너 본인 철학 명확하면 기존 방식. 비디자이너는 신규에서도 유용 |
| Manus·Lovable과 연결? | GitHub을 통해 가능 |
| 이미지 던지고 프론트엔드 디자인 vs Claude Design? | **100% Claude Design** — 화면이 늘어날수록 일관성 보장 |
| 펜슬이 Claude Design보다 나은 점? | 펜슬은 화면별 디자인 파일 → Claude Code가 정확하게 구현. 디테일 + 토큰 절약 |

### 3-10. 인준님 권장 워크플로우 (현업 적용)

```
① Claude Design  →  ② 펜슬  →  ③ Claude Code
   (디자인 시스템     (화면별        (펜슬 디자인을
    추출)            리디자인)      컨텍스트로 구현)
```

이렇게 하면 *"AI 냄새"* (간격 들쭉날쭉, 버튼 제각각)를 상쇄하고 **일관된 완성도**를 얻을 수 있음.
